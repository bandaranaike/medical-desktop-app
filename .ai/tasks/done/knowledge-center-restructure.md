## Goal

Turn `.ai/` into a structured knowledge center with clearer document grouping, a multilingual task inbox, and a task status workflow that future agent work can follow consistently.

## Scope

- organize `.ai/` into clearer sections for guidance, references, and task tracking
- create an obvious place where the user can drop raw task ideas in any language
- add a task registry and status definitions so task progress can be tracked
- update repo guidance so future work reads the new structure first
- move existing task files into a structured status area without falsely marking unknown work as done

## Constraints

- preserve the actual content of existing task notes unless a small metadata update is needed
- avoid inventing completion status for older task files whose implementation state is unclear
- keep the new workflow simple enough that the user can maintain it without extra tooling

## Done When

- `.ai/` has a clearer structure with documented entry points
- there is a user-facing inbox file for raw task notes in any language
- there is a documented task status flow and a registry file
- repo guidance points future work to the structured `.ai/` layout
